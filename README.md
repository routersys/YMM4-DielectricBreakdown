# 誘電破壊 for YMM4

[![License](https://img.shields.io/badge/license-MIT-blue.svg)](#)
[![.NET](https://img.shields.io/badge/.NET-10.0-purple.svg)](#)
[![Release](https://img.shields.io/github/v/release/routersys/YMM4-DielectricBreakdown.svg)](https://github.com/routersys/YMM4-DielectricBreakdown/releases)

---

YukkuriMovieMaker4（YMM4）上で動作する、素材の不透明な部分を電極とみなし、誘電絶縁破壊モデルで放電経路を成長させて稲妻として描画する映像エフェクトプラグインです。
履歴フレーム、参照画像、学習済みモデルを使わず、入力された一枚の映像だけを処理します。
放電経路はシードから決定論的に決まり、同じ設定では常に同じ形になります。
強さ、成長、方向、距離、分岐、太さ、発光の7項目はアニメーションに対応しています。

## 概要

本プラグインはYMM4の映像エフェクトとして動作し、エフェクトの種類一覧では「誘電破壊」として表示されます。カテゴリは装飾とアニメーションです。

入力された現在のフレームのアルファ値から素材の輪郭を求め、輪郭のセルを電極として成長格子の上に置きます。電極からの電界ポテンシャルと方向バイアスに従い、放電経路を1ステップに1セルずつ確率的に成長させます。成長先の選択は、ポテンシャルとシードから決まるスコアの最大値で行うため、同じシードと同じパラメータでは常に同じ経路になります。

成長は、方向が定める向きへ距離の分だけ進んだ時点で対象へ到達したとみなして停止します。成長のパラメータは、この経路をどこまで表示するかを決めます。100%で経路が対象へ到達し、主経路が主放電として強く光ります。求めた経路は、太さを持つ芯と、周囲に広がる発光として描画し、元映像へスクリーン合成します。

計算はComputeSharpの計算シェーダーがDirect3D 12で実行します。YMM4のDirect3D 11側とComputeSharpのDirect3D 12側は、共有テクスチャと共有フェンスで接続します。通常のフレーム処理では、CPUへの画素の読み戻しを行わず、成長の記録だけを読み戻します。稲妻が存在する矩形の範囲だけを描画し、素材の形状とパラメータが変わらないフレームでは成長の計算を再利用します。

このエフェクトはAviUtl向けのEXO出力には対応していません。

## インストール方法

1. [Releases](https://github.com/routersys/YMM4-DielectricBreakdown/releases/latest)ページから最新のプラグインファイル（`.ymme`）をダウンロードしてください。
2. YMM4が起動していないことを確認し、ダウンロードしたファイルを実行してインストールします。
3. YMM4を起動し、タイムライン上のアイテムに映像エフェクトを追加します。
4. 映像エフェクトの種類として「誘電破壊」を選択してください。

## 主な機能

### 1. 放電経路の成長

現在のフレームのアルファ値から素材の輪郭を求め、輪郭のセルを電極として成長格子の上に置きます。電極が作る電界ポテンシャルと、方向が定める外部電界を合わせた場の中で、構造に隣接する空きセルから成長先を1ステップに1セルずつ選びます。成長したセルは新しい電荷として場へ加わるため、経路が伸びるほど先端へ電界が集中します。

外部マスクや手動の位置指定は必要としません。成長は品質設定が定めるステップ数を上限として、距離に応じた回数だけ進みます。

### 2. 成長と主放電

成長は、求めた放電経路をどこまで表示するかを決めます。0%では何も描画せず、値を上げると経路が電極から先端へ向かって伸びていきます。伸びている間は先端が強調されて光ります。

100%で経路が対象へ到達し、電極から到達点までの主経路が主放電として強く光ります。成長を時間で変えると、稲妻が伸びて落ちるまでの遷移になります。

### 3. 方向と距離

方向は、稲妻が伸びる向きを決めます。0°で下方向へ伸び、角度に応じて任意の向きへ成長します。

距離は、対象までの距離を素材の長辺に対する割合で決めます。成長が方向の向きへ距離の分だけ進むと、対象へ到達したとみなして成長を停止します。距離を大きくするほど計算領域が広がり、処理量が増えます。

### 4. 分岐とシード

分岐は、枝分かれの密度を決めます。低いほど一本の経路に集中し、高いほど樹枝状に広がります。

シードは、放電経路を決める固定値です。経路の選択と先端の揺らぎは、シードとセル位置から決定論的に決まります。同じシードと同じパラメータでは、常に同じ経路になります。

### 5. 太さ・発光・色

太さは、稲妻の芯の太さを画素単位で決めます。主経路は太く、枝は細く描画されます。

発光は、稲妻の周囲に広がる光の強さを決めます。発光は経路に沿って堆積した光を1/4解像度の格子の上でぼかして作ります。

色は、稲妻の色を決めます。芯は白く発光し、周囲は指定した色で光ります。

### 6. 品質の選択

品質は、成長格子の解像度と成長ステップ数の上限をまとめて切り替えます。

| 品質 | 格子解像度 | 成長ステップ上限 |
|---|---:|---:|
| 標準 | 144 | 640回 |
| 高品質 | 192 | 1024回 |
| 最高品質 | 256 | 1536回 |

格子解像度は計算領域の長辺のセル数です。短辺のセル数は計算領域の縦横比に合わせて決めます。実際のステップ数は距離に応じて決まり、上限で打ち切ります。

## パラメータ一覧

### 基本グループ

| パラメータ名 | 型 | デフォルト | スライダー表示範囲 | アニメーション | 説明 |
|---|---|---|---|---|---|
| 強さ | 数値 | 100% | 0〜100% | 対応 | 稲妻の合成量です。0%では入力を変更しません。 |
| 成長 | 数値 | 100% | 0〜100% | 対応 | 放電経路の伸び具合です。100%で対象へ到達し主放電が光ります。 |
| 品質 | 列挙 | 高品質 | 標準 / 高品質 / 最高品質 | 非対応 | 成長格子の解像度と成長ステップ数を選択します。 |

### 放電グループ

| パラメータ名 | 型 | デフォルト | スライダー表示範囲 | アニメーション | 説明 |
|---|---|---|---|---|---|
| 方向 | 数値 | 0° | -180〜180° | 対応 | 稲妻が伸びる方向です。0°で下方向に伸びます。 |
| 距離 | 数値 | 75% | 10〜200% | 対応 | 対象までの距離です。素材の長辺に対する割合で指定します。直接入力では1〜400%まで指定できます。 |
| 分岐 | 数値 | 50% | 0〜100% | 対応 | 枝分かれの密度です。高いほど樹枝状に広がります。 |

| パラメータ名 | 型 | デフォルト | スライダー表示範囲 | アニメーション | 説明 |

| 発光 | 数値 | 50% | 0〜100% | 対応 | 稲妻の周囲に広がる光の強さです。 |

## 制限事項

- AviUtl向けのEXO出力には対応していません。
- Direct3D 12デバイスを取得できない場合や、YMM4が使うDirect3D 11デバイスとの共有テクスチャおよび共有フェンスを作成できない場合は、エフェクトを適用せず入力映像を表示します。
- 電極の判定には現在のフレームのアルファ値だけを使用します。輝度と色は判定に影響しません。履歴フレーム、外部画像、学習済みモデルは使用しません。
- 計算領域は素材の範囲に距離と発光の余白を加えた大きさで、長辺は8192画素が上限です。素材が大きく余白を確保できない場合は、エフェクトを適用せず入力映像を表示します。
- 処理量とGPUメモリ使用量は、計算領域の大きさと品質に応じて増えます。

## 注意事項

- 処理負荷: 最高品質は格子解像度と成長ステップ数が最も多いため、高品質と標準より処理負荷が高くなります。距離を大きくするほど計算領域が広がります。描画は稲妻が存在する矩形の範囲だけを計算します。
- キャッシュ: 素材の輪郭と経路を決めるパラメータが変わらないフレームでは、成長の計算を再利用します。成長、太さ、発光、色だけを変えたフレームでは、経路を再計算しません。
- 成長0%と強さ0%: 成長が0%のときは何も描画しません。強さが0%のときは入力を変更しません。
- シードと再現性: 放電経路は、シード、セル位置、成長ステップから決定論的に算出します。時間値は経路の生成に使用しません。
- 透明部分: 稲妻は素材の外側にも描画されるため、出力範囲は素材より広がります。稲妻は元映像へスクリーン合成で加算します。
- 更新通知: エフェクトを最初に生成したときに、YMM4の実行中に一度だけ新しい正式版を確認します。更新情報を取得できない場合は通知せず、そのまま動作します。プレリリースは通知の対象に含めません。
- 本プラグインを使用する前に、YMM4のプロジェクトファイルのバックアップを作成することを推奨します。
